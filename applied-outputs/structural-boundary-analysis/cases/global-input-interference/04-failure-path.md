## Failure Path and Recurrence

The failure is not tied to a single application.
Any software relying on modifier-based input distinctions is susceptible.

Because the interception operates on wildcard rules:
- new applications can be affected without code changes,
- updates do not guarantee resolution,
- and similar failures may emerge silently.

This represents a recurring structural failure path, not an isolated defect.

## 실패 경로와 반복성

이 실패는 특정 애플리케이션에 한정되지 않는다.
수정자 키 기반 입력 구분에 의존하는 모든 소프트웨어가 영향을 받을 수 있다.

전역 와일드카드 규칙으로 동작하기 때문에:
- 애플리케이션 변경 없이도 새로운 피해가 발생할 수 있고,
- 업데이트가 해결을 보장하지 않으며,
- 유사한 실패가 조용히 반복될 수 있다.

이는 단일 결함이 아니라 반복 가능한 구조적 실패 경로다.
