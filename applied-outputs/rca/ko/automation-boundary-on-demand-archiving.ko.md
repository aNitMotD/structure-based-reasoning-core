## Case: On-Demand Archiving under Automation Boundary

### 1. 관측 가능했던 정보
- Windows 환경에서 비동기 엔진(Playwright) 실행 시 무작위 프로세스 크래시 발생
- greenlet C-extension과 Windows 바이너리 링킹 방식 간 충돌 확인
- 외부 미들웨어(BGUtil) 포트 충돌 및 좀비 프로세스 잔존 현상 관측
- 세션 만료 및 인증 실패가 반복적으로 발생

### 2. 암묵적으로 요구된 판단
- 런타임 불안정성을 운영상의 일시적 문제로 취급하고 지속 운영할 것
- 포트 충돌을 수동 개입으로 관리할 것
- 인증 만료를 사용자 책임 또는 반복 작업으로 처리할 것

### 3. 판단 불가능성의 구조적 원인
- 비동기 엔진의 불안정성은 운영 차원에서 통제 불가
- 고정 포트 기반 미들웨어는 OS 예약 대역과 구조적으로 충돌 가능
- 인증 만료를 인간 개입에 의존할 경우 Zero-Touch 요건 붕괴

### 4. 구조적 판단 전환
- C-level 의존성을 제거하고 동기 기반 Selenium으로 전환
- 런타임 포트를 동적으로 할당하는 구조로 변경
- 인증 만료를 인간 판단이 아닌 시스템 자가 치유 영역으로 이동

### 5. 책임 경계 선언
- 런타임 안정성과 인증 지속성은 인간 운영 판단의 영역이 아님
- 해당 영역은 시스템 설계 단계에서 완결되어야 함

### 6. 사후 통찰
- 본 케이스는 **‘운영 중 판단을 요구하는 자동화 구조’가 실패하는 전형 사례**
- 인간 개입 여부가 아니라 **판단을 요구하지 않도록 설계했는가**의 문제
