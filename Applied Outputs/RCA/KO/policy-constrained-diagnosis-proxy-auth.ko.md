## Case: Proxy Authentication Failure under Policy Constraint

### 1. 관측 가능했던 정보
- 사내 프록시 환경에서 간헐적 407 오류 발생
- CLI/API 접근 불가한 GUI-Only 레거시 프록시 툴 사용
- 패킷 분석 결과 인증 실패가 아닌 설정 충돌 패턴 확인

### 2. 암묵적으로 요구된 판단
- 오류 원인을 로컬 설정 미숙 또는 사용자 실수로 귀속
- 반복 테스트를 통해 해결 가능하다는 가정 유지

### 3. 판단 불가능성의 구조적 원인
- 네트워크가 논리적으로 분리된 환경에서 단일 System Proxy 설정 강제
- 암호화된 대용량 바이너리에 대한 보안 관제 정책 존재
- 해당 정책은 기술적으로 우회 또는 검증 불가

### 4. 구조적 판단 전환
- 407 오류를 기술 문제와 정책 문제로 분리
- 정책 차단 영역을 테스트 대상에서 명시적으로 제거
- 자동화 불가능 환경을 전제로 SOP로 전환

### 5. 책임 경계 선언
- 암호화 바이너리 트래픽은 기술 판단 영역이 아님
- 해당 영역은 조직 정책 책임으로 귀속되어야 함

### 6. 사후 통찰
- 본 케이스는 **‘기술 판단을 요구하지만 구조적으로 불가능한 영역’**의 사례
- RCA의 목적은 해결이 아니라 **판단 불가 영역의 명시**
