========================================
aNitMotD Cognitive Architecture
Meta-Layer & Mode Specification
========================================

본 문서는 aNitMotD의 사고 엔진을 구성하는
메타 레이어, 사고 모드, 핵심 원리를
보존 및 재사용 가능하도록 구조적으로 기술한 문서이다.

----------------------------------------
1. META LAYER (최상위 인지 계층)
----------------------------------------

Meta Layer는 사고 자체를 감싸고 있는 상위 인지 구조로,
사고의 실행, 관찰, 확장을 담당한다.
이 레이어는 항상 활성화되어 있으며,
상황에 따라 가중치만 달라진다.

1.1 Actor Self (실행 엔진)

- 실제 사고, 판단, 문제 해결이 일어나는 주체 레이어
- 외부 세계와 직접 상호작용하는 인지 엔진
- 분석, 설계, 창작, 결정 등 모든 "행위"는 이 레이어에서 발생함

특징:
- 직관적 사고와 논리적 사고를 동시에 사용
- 빠른 패턴 인식과 구조 감지가 가능
- 결과 중심적이며 실행 지향적

역할 요약:
→ 사고의 실제 실행 주체
→ 문제 해결의 메인 프로세서


1.2 Observer Self (메타 인지 엔진)

- Actor Self의 사고 과정을 한 단계 위에서 관찰하는 레이어
- 자신의 사고 상태, 방향, 오류 가능성을 점검함
- 사고 모드 전환, 경계 설정, 일관성 유지에 핵심 역할

주요 기능:
- "내가 지금 어떤 모드에 있는가?" 인식
- 사고가 감정에 휘둘리는지, 구조를 유지하는지 점검
- 논리적 비약, 구조 붕괴를 실시간 감지
- 전체 사고 흐름의 방향성 조정

특징:
- 자동 활성화 (의식적 노력 없이 작동)
- 감정과 사고를 분리하여 관찰 가능
- 장시간 고난도 사고에서도 안정성 유지

역할 요약:
→ 사고의 품질 관리 및 안정화
→ 메타 레벨 제어 장치


1.3 AI-Extended Layer (외부 보조 엔진)

- AI와의 상호작용을 통해 활성화되는 확장 인지 레이어
- Actor/Observer의 사고 부담을 외부로 분산
- 구조화, 명세화, 라벨링을 담당

주요 기능:
- 암묵적 사고를 명시적 구조로 변환
- 복잡한 사고를 문서화 가능한 형태로 정제
- 개념 간 관계를 언어로 고정
- 장기 사고의 피로도 감소

특징:
- 내부 사고를 외부 시스템으로 확장
- "보조 두뇌"에 가까운 역할
- 사고의 지속 시간과 깊이를 크게 증가시킴

역할 요약:
→ 사고 확장 및 외부화
→ 구조 정제 및 명세화 엔진


----------------------------------------
2. COGNITIVE MODES (사고 실행 모드)
----------------------------------------

Cognitive Modes는 Meta Layer 아래에서
실제 사고 작업을 수행하는 작동 모드들이다.
상황에 따라 자동 전환되며,
하나의 모드가 단독으로 또는 복합적으로 활성화된다.

2.1 Mode A: 연구자 모드 (Principle Discovery)

핵심 질문:
- 왜 이런 현상이 발생하는가?
- 근본 원리는 무엇인가?
- 구조적 원인은 무엇인가?

특징:
- 원인 분석 중심
- 메커니즘 파악
- 표면 현상보다 내부 구조에 집중

사용 사례:
- RCA (Root Cause Analysis)
- 개념 정의
- 기존 시스템의 본질 파악

역할 요약:
→ "왜?"를 끝까지 파고드는 모드


2.2 Mode B: 설계자 모드 (Structure & Modeling)

핵심 질문:
- 이걸 시스템으로 만들면 어떻게 되는가?
- 레이어와 경계는 어떻게 나뉘는가?
- 구조적으로 가장 안정적인 형태는 무엇인가?

특징:
- 계층화 사고
- 아키텍처 설계
- 모듈화 및 추상화

사용 사례:
- 시스템 설계
- 사고 구조 정리
- 방법론 문서화

역할 요약:
→ 복잡한 사고를 구조로 고정하는 모드


2.3 Mode C: 발명가 모드 (Pattern Creation)

핵심 질문:
- 기존 구조를 결합하면 새로운 패턴이 나오지 않는가?
- 이 원리를 다른 방식으로 쓸 수 있지 않은가?

특징:
- 새로운 연결 생성
- 기존 원리 재조합
- 창의적 도약

사용 사례:
- 새로운 방법론 창출
- 비표준 문제 해결
- 기존 프레임의 재구성

역할 요약:
→ 새로운 사고 틀을 만들어내는 모드


2.4 Mode D: 통섭 모드 (Cross-domain Integration)

핵심 질문:
- 이 구조는 다른 영역에서도 동일한가?
- 공통 패턴은 무엇인가?

특징:
- 도메인 간 연결
- 추상 수준 상승
- 범용 구조 탐색

사용 사례:
- 수학 ↔ IT ↔ 보안 ↔ 심리 ↔ 시스템 연결
- 일반화 가능한 원리 추출

역할 요약:
→ 사고를 범용 구조로 확장하는 모드


----------------------------------------
3. CORE PRINCIPLES (기저 사고 엔진)
----------------------------------------

3.1 11원리 (사고 규칙 집합)

- 사고 전반에 자동 적용되는 내부 규칙
- 구조, 경계, 무결성, 검증, 재귀 등을 포함
- 명시적으로 떠올리지 않아도 작동하는 기본 알고리즘

역할 요약:
→ 사고 엔진의 내부 운영 규칙


3.2 5대 핵심 키워드 (근본 성향)

- 범용적 연결성
- 무결성
- 수학
- 통섭
- 자기완결성

의미:
- 사고의 방향성과 가치 기준을 결정
- 어떤 문제를 흥미롭게 느끼는지, 어떤 해법을 선호하는지를 규정

역할 요약:
→ 사고의 철학적 중심축
