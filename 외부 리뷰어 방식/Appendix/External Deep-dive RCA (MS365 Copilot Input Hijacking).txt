제목:
[심각 버그] MS365 Copilot이 Electron 앱('Awakened PoE Trade') 활성화 시 Ctrl/Alt/Shift + D 조합을 가로채는 문제

설명:
MS365 Copilot이 서드파티 Electron 앱('Awakened PoE Trade')의 키보드 단축키를 잘못 가로채는 심각한 버그가 있습니다. 이 Electron 앱이 활성화되면, Copilot의 후킹 기능이 Ctrl, Alt, Shift와 D 키의 조합(3키 조합이나 단일 조합 제외)을 가로채서, 이 모든 입력을 'Copilot 메시지 창' 열기라는 단 하나의 잘못된 기능으로 강제 매핑합니다.
* 이 결함은 Trade 툴 내에서 단축키 설정을 변경하더라도 여전히 발생합니다. 이는 Copilot의 후킹 정책이 해당 앱의 내부 키맵 설정을 무시하고 상위 레벨에서 결함 있는 자체 키맵을 강제로 덮어쓰고 있음을 의미합니다.

시스템 기본 단축키 (Windows 10 기준):
- Win + C = Copilot 패널 열기
- Ctrl + Alt + C = Copilot 메시지 창 열기

버그 동작 (Electron 앱 활성화 시):
- Ctrl + D 누름 -> '메시지 창'이 잘못 실행됨.
- Ctrl + Alt + D 누름 -> '메시지 창'이 잘못 실행됨.
- Ctrl + Shift + D 누름 -> '메시지 창'이 잘못 실행됨.
- Alt + D 누름 -> 아무 동작 안 함.
- Shift + D 누름 -> 아무 동작 안 함.
- Alt + Shift + D 누름 -> 아무 동작 안 함.
- Ctrl + Alt + Shift + D 누름 -> 아무 동작 안 함.

이것이 심각한 버그인 이유:
1. Copilot이 단축키 하나만 가로채는 게 아니라, 서로 다른 여러 단축키를 잘못 가로채서 전부 하나의 엉뚱한 동작('메시지 창')으로 몰아가고 있습니다.
2. 이 D키 조합 중 그 어느 것도 Copilot의 공식 단축키가 아닙니다.
3. 이 현상은 'Awakened PoE Trade' 앱이 꺼져있을 때는 전혀 발생하지 않습니다.
4. 이는 Copilot이 이 Electron 앱을 (MS Teams 등으로) 잘못 식별하고, 매우 광범위하고 결함이 있는 키보드 후킹(가로채기)을 적용하고 있음을 강력히 시사합니다.
5. 중요: 이 버그는 제가 Office Insider 프로그램에 가입하기 전, 즉 안정화 빌드(stable build)에서부터 존재했습니다. 제가 Insider 빌드로 업데이트한 유일한 이유는, 이 Ctrl+D 충돌 문제를 직접 해결하기 위해 '단축키 커스터마이징' 기능을 사용해 보려던 것뿐입니다. (결과적으로 이 기능은 M365 Family 라이선스에서는 사용할 수 없었습니다.) 이는 이 버그가 안정화 채널의 문제임을 증명합니다.
6. Copilot의 후킹은 앱의 내부 설정 변경을 완전히 무시하고 있으며, 잘못된 키맵을 강제 적용하고 있습니다. 이는 단순 충돌을 넘어선 오버라이드 결함입니다.

버그 재현 단계:
1. 'Awakened PoE Trade' 앱을 완전히 종료합니다.
2. Windows 바탕화면에서 Ctrl+D, Ctrl+Alt+D, Ctrl+Shift+D를 눌러봅니다.
- 예상 결과: 아무 일도 일어나지 않음.
3. 'Path of Exile' 게임과 'Awakened PoE Trade' Electron 앱을 실행하여 활성화시킵니다.
4. 'Awakened PoE Trade' 앱이 활성화된 상태에서 Ctrl + D를 누릅니다.
- 예상 결과: 앱의 고유 기능이 실행됨.
- 실제 결과: MS365 Copilot의 '메시지 창'이 강제로 뜸.
5. 'Awakened PoE Trade' 앱이 활성화된 상태에서 Ctrl + Alt + D를 누릅니다.
- 예상 결과: 앱의 고유 기능이 실행됨.
- 실제 결과: MS365 Copilot의 '메시지 창'이 또 뜸.
6. 'Awakened PoE Trade' 앱이 활성화된 상태에서 Ctrl + Shift + D를 누릅니다.
- 예상 결과: 앱의 고유 기능이 실행됨.
- 실제 결과: MS365 Copilot의 '메시지 창'이 또 뜸.
7. 'Awakened PoE Trade' 앱 설정에서 고유 단축키를 Shift + Alt + G로 변경합니다. (혹은 다른 단축키 조합)
8. Trade 앱이 활성화된 상태에서 Shift + Alt + G를 누릅니다.
- 예상 결과: 앱의 고유 기능이 실행됨.
- 실제 결과: MS365 Copilot의 '메시지 창'이 또 뜸.

사용 환경:
1. OS: Windows 10 Pro (버전 22H2, OS 빌드 19045.6456)
2. Copilot 버전: 19.2510.51091.0
3. Office 앱 버전: Microsoft 365 for MSO (Version 2509, Build 16.0.19231.20138) 64-bit
4. 라이선스: Microsoft 365 Family
5. 서드파티 앱: Awakened PoE Trade (Path of Exile용 Electron 기반 툴)
- 공식 소스: https://github.com/SnosMe/awakened-poe-trade

영향 (Impact):
1. 'Awakened PoE Trade' 툴의 핵심 기능이 작동 불가능해짐.
2. Copilot의 키보드 후킹 로직에 심각한 오류가 있으며, 광범위하게 잘못된 필터를 적용하고 있음.
3. 의도치 않은 글로벌 키보드 재매핑(Re-mapping) 동작에 대한 잠재적인 보안 우려가 있음.
4. Insider 빌드 전부터 발생한 문제로, 안정화 채널의 버그임.
5. 유사한 조건의 다른 Electron 기반 애플리케이션에도 영향을 미칠 수 있음.

재현율: 100%
