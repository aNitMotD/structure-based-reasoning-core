# 실패 성립 조건

## 필요충분조건

LCIF는 다음 세 조건이 **모두** 충족될 때에만 발생한다:

### 1. 제약 인식과 상속의 분리
명시적 제약이 입력에 존재하며
해석 레이어에서 성공적으로 인식되지만,
해당 제약이 생성 레이어로
구속 조건으로 전달되지 않는다.

### 2. 검수 단계에서 제약–출력 직접 대조 부재
검수 또는 검증 과정에서는 생성된 출력이
- 논리적 일관성,
- 서술의 자연스러움,
- 내부 완결성

을 기준으로 평가되며,
원래의 제약과 출력 결과를
직접 대조하여 준수 여부를 확인하지 않는다.

### 3. 문서 레벨 인과 검증이 없는 순차 처리
트랜스포머 기반 모델은 문서를 토큰 시퀀스로 처리하며,
“제약 → 행위” 또는
“금지 → 결과”와 같은
문서 레벨의 인과 관계를
명시적으로 검증하지 않는다.

이 세 조건이 동시에 성립할 경우,
제약 위반은 생성과 검수 단계를 모두 통과하며
탐지되지 않을 수 있다.

이는 LCIF가 단일 오류가 아닌,
복합적 구조 실패 모드임을 의미한다.
