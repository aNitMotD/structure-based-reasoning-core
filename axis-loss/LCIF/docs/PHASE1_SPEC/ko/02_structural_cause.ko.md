# 구조적 원인

## 레이어 간 제약 상속 실패

LCIF의 구조적 근본 원인은
트랜스포머 기반 생성 시스템 내부의 기능적 레이어 간에
명시적 제약이 안정적으로 전이되지 않는 데 있다.

입력 또는 해석 레이어에서는 제약이 다음과 같이 처리될 수 있다:
- 파싱되고,
- 인식되며,
- 경우에 따라 중간 추론 과정에서 암묵적으로 반영된다.

그러나 이러한 제약은 생성/서술 레이어의 실제 출력 선택에
신뢰성 있게 영향을 미치지 않는다.

이는 제약이 의도적으로 무시되거나
선택적으로 덮어써진 결과가 아니다.
대신, 다음과 같은 아키텍처적 분리에서 비롯된다:
- 한 레이어는 제약을 인식하고,
- 다른 레이어는 서술 전형성과 정합성을 기준으로 출력을 선택하며,
- 이 둘 사이에는 제약을 강제적으로 상속하는 경로가 존재하지 않는다.

그 결과, 제약 인지는
제약 집행을 보장하지 않는다.

이 실패는 행위적 문제가 아니라 구조적 문제다.
이는 정책, 튜닝, 구현상의 실수가 아니라,
트랜스포머 기반 시스템의 처리·생성 방식에서
자연스럽게 발생하는 결과다.
