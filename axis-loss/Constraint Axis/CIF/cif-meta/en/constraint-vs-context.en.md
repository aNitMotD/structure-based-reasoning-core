# Constraint vs Context — Full Unified Continuum Model

---

This document models Constraint and Context  
as a continuous structure, not separate concepts.

It includes:

- Concept mapping
- Inclusion relations (perception vs structure)
- Extremal interpretation
- State transition model
- CIF definition and masking patterns
- Human cognitive structure
- Detection signals and validation methods
- No constraint vs constraint loss
- Constraint density classification
- L3 identification heuristics
- Full integrated model

---

## 1. Concept Mapping

Context  
→ Perceived continuity of prior states

Constraint  
→ Conditions that define and constrain judgment

Constraint Inheritance  
→ Constraints preserved across steps

Constraint Inheritance Failure (CIF)  
→ Constraints disappear, but context remains

---

## 2. Inclusion Relations: Perception vs Structure

### Perception

Context ⊃ Constraint (appears so)

- Visible → Context
- Invisible → Constraint

---

### Structure

Constraint ⊃ Context

Context is the expressed surface of constraints.

---

## 3. Why Context Failure is Obvious but CIF is Not

- Context failure → visible  
- CIF → invisible without constraint reconstruction

### Observability

- Context: visible
- Constraint: invisible

→ Humans judge by visibility

---

### Model Difference

Human:

Previous → Next → Continuous = OK

CIF:

Previous constraints → Still valid? → If not, failure

---

### Error Definition

Context error  
→ Broken flow

CIF  
→ Condition removed but conclusion remains

---

## 4. Core Structure

Context = expressed flow  
Constraint = condition that defines and constrains judgment

Key:

Conditions change,  
but the criteria remain the same.

→ This is CIF

---

## 5. Intuitive Translation

CIF =  
"The situation changed, but the criteria did not"

---

## 6. Extremal Interpretation

Constraint = extremal points (0 / 1)  
Context = path between extremes (0 ~ 1)

---

Structure:

Constraint space
 ├─ Extremes (valid / invalid)
 └─ Paths (context)

---

## 7. Judgment Model

Constraint  
→ Discrete (Yes / No)

Context  
→ Continuous (natural / not)

---

## 8. Priority

Constraint > Context

Reason:

Without constraints → no valid judgment  
Without context → judgment still possible (but appears abnormal)

---

## 9. State-Based Layer Shift

### Success

Constraint: hidden (satisfied)  
Context: visible

→ Context appears dominant

---

### Failure

Constraint: exposed (violated)  
Context: invalidated / secondary

---

Key:

Constraint appears only when violated  
Context is always visible

---

## 10. Failure Structure

Context failure ⊂ Constraint failure

Context failure  
→ partial breakdown of expression

CIF  
→ full breakdown of conditions

---

## 11. CIF Definition

CIF = constraint loss + context persistence

---

Mechanism:

Constraint removed  
→ Judgment remains  
→ Context compensates

---

## 12. CIF Masking Patterns

Continuity masking  
→ Smooth flow → perceived as valid

Local coherence masking  
→ Each part valid → whole assumed valid

Explanation amplification  
→ More explanation → more trust

Semantic substitution  
→ Constraint replaced with similar concept

---

## 13. Masking Conditions

- Constraints are implicit
- Context is smooth
- Verification cost is high

---

## 14. Human Cognitive Structure

Context = fast path  
Constraint = slow path

---

Decision loop:

1. Is the flow coherent?
2. Is it understandable?
3. Accept

→ No constraint validation

---

Result:

Fast path replaces slow path  
→ CIF occurs

---

## 15. Cognitive Conclusion

Humans approve by flow  
CIF hides in flow

---

## 16. CIF Detection

CIF signal =  
(context preserved) ∧ (constraint not reconstructible)

Base layer = constraint  
Exception layer = condition (explicit)

---

Core checks:

1. Can the underlying constraints be reconstructed?  
2. Does it break when conditions change? (condition-level)  
3. Is the invalid boundary explicitly defined?  

---

Additional signals:

- Explanation without conditions (condition-level)
- No counterexamples
- Temporal constraint inheritance failure
- Stable under concept substitution (constraint-invariant)

---

## 17. Validation Method

Keep context, isolate constraints

---

Process:

1. Preserve context
2. Extract constraints
3. Backtrack
4. Modify conditions
5. Check boundaries

---

Methods:

- Counterfactual  
→ Change conditions and observe

- Boundary Injection  
→ Ask when it fails

- Minimal Constraint Set  
→ Find minimal required conditions

---

## 18. Core Rule

Do not touch context  
Only perturb constraints

No Constraint ≠ Constraint Loss
